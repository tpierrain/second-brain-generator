## Context

All v3.3.0 features (**F1→F8**) and the **8 `/code-review` findings** are **shipped green, committed,
NOT pushed** on branch `v3.3.0-self-converging-brain-and-citations`. The only thing standing between the

This document is the **execution plan for that gate**, split so the work is parallelizable:

  toasts): the things that, by construction, only a human in front of the app can observe.

is a hard gate (ADR 0015); no CI → suites run locally; **a red QA item that needs a code fix is fixed in

explicit green light only.

### A2 — F1 headless upgrade replay (the centerpiece, mechanism-level) 🤖

The TRUE single-pass UX proof is human (B1). But the **mechanism** is fully checkable headless against the
pristine tarball. One disposable pass:

- [ ] `rm -rf /tmp/qa-legacy && mkdir -p /tmp/qa-legacy && tar -xzf ~/legacy-brain.tgz -C /tmp/qa-legacy`
  → a fresh v3.1.0 brain (rag 1.1.0) at `/tmp/qa-legacy/legacy-brain`. **Never extract onto `~/legacy-brain`
  for the headless pass** (that path is reserved for B1's real Desktop run); use `/tmp` so the two never collide.
- [ ] Snapshot the sacred surface BEFORE: hash `vault/`, `.env`, `CLAUDE.md`, `.claude/settings.json`, and the
  user-portion of `.mcp.json` (e.g. `find vault -type f -exec shasum {} + | sort` + `shasum` of the four files).
- [ ] Drive the converge headless: run the reconciler CLI against the extracted brain with `sourceDir` = this
  launcher (`node scripts/reconcile-brain.mjs --brainDir /tmp/qa-legacy/legacy-brain --sourceDir $PWD` — the
  same entry auto-finalize re-execs). Assert in **one** pass: the new engine skill dir(s) exist, `local-mirror`
  + `vault-rag` are present in the brain `.mcp.json`, launchers regenerated.
- [ ] **Idempotence:** run the same command a **second** time → assert zero churn (no files rewritten; `copied`
  empty; `.mcp.json` byte-identical).
- [ ] **F1 safety invariant:** re-hash the sacred surface AFTER both runs → assert `vault/` (minus the seeded
  `vault/engine-health/health-check.md`), `.env`, `CLAUDE.md`, `settings.json`, and any user-added `.mcp.json`
  server are **byte-identical** to the BEFORE snapshot. The ONLY allowed new vault file is the engine-health note.
- [ ] **F7-bis upgrader carve-out:** assert `vault/engine-health/health-check.md` is seeded on the upgrader
  (it was absent in the v3.1.0 tarball) and that a **second** reconcile does NOT re-seed / re-reindex it
  (`reindexReason` absent on run 2).
- [ ] `rm -rf /tmp/qa-legacy` when done.

> Honest scope of A2: this proves the **reconciler converges + is idempotent + leaves the vault sacred**. It
> does **not** prove the Node-module-cache "Layer A" auto-finalize child escape in a live `update-engine`
> process, nor the "config-freeze" capability pickup — those are B1/B2 (a running Desktop).
