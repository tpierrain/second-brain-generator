## Per-finding detail (file:line + fix + test)

### #1 — `scripts/update-engine.mjs:173` + `scripts/lib/auto-finalize.mjs:32` 🔴
**Bug:** `updateEngine` writes the new manifest (step 7 = update done + recorded), then `await
finalizeReconcile(...)` at step 8 with **no try/catch**. `defaultFinalizeReconcile` uses `execFileSync`
(throws on non-zero child exit). A finalize-child failure (flaky npm install in the fresh process, ABI
hiccup) propagates to the CLI `.catch` (line 190) → prints **"❌ update-engine failed — the brain was
NOT changed past this point"** + exit 1, even though the update fully succeeded. Violates auto-finalize's
own docstring (*"never throwing past update-engine's own success"*) and is actively misleading.
**Fix:** make finalize fail-soft. Either wrap the call (`try { await finalizeReconcile(...) } catch (e) {
report a soft "couldn't auto-finalize, restart/run once more" note }`) OR give `defaultFinalizeReconcile`
the try/catch its docstring promises (return `{ finalized:false, error }` instead of throwing). Prefer the
production seam carrying the contract so every caller is safe.
**Test:** `scripts/lib/auto-finalize.test.mjs` / `update-engine.test.mjs` — inject a `finalizeReconcile`
(or `spawnChild`) that throws; assert `updateEngine` still RESOLVES with the recorded report (does not
reject), and that `formatReport`/the CLI never claims "NOT changed". RED first.

### #2 — `scripts/lib/self-heal-detect.mjs:18` 🟠
**Bug:** reads `manifest.installSkills ?? []`, but the real `engine-manifest.json` has NO top-level
`installSkills` key (skills live under `regimes.merge`; `installSkills` is derived only by
`computeApplyPlan()`). `session-self-heal.mjs:72` passes the raw parsed manifest → `missingSkills` is
ALWAYS `[]`. The unit test masks it by injecting a fabricated top-level `installSkills`.
**Fix:** in `detectSelfHealGap` (or its wrapper), derive the skill list via `computeApplyPlan(manifest).installSkills`
(same source `reconcileBrain` uses), then `skillGlobToDir` + `skillDirExists`. Keep the function pure/injectable.
**Test:** fix `self-heal-detect.test.mjs` to pass a REAL-shaped manifest (skills only under `regimes.merge`)
and assert a missing engine skill IS detected. Confirm it fails against the current code first.
**Note:** also sanity-check `reconcile-brain.mjs:89` — in self-heal `sourceDir === brainDir`, so a genuinely
missing skill has no source to copy from. Detection is still worth fixing (honest diagnostic + future-proof),
but document that the runtime self-heal can register a missing MCP server but cannot materialise skill files
that aren't on disk (auto-finalize Layer A remains their real fix). Decide with Thomas whether to keep the
skill branch as a detector-only or drop it — do NOT silently leave a dead field read.

### #3 — `scripts/lib/health-check-gate.mjs:17` 🟠
**Bug:** `gateBlockers` blocks on ANY `broken` (no optional carve-out). The installer post-flight +
verify-rag round-trip every registered module including optional `local-mirror`; if its server can't boot
(its npm install only `warn`s at installer.mjs:736, no exit), `local-mirror` → `broken` → exit 1, failing
an otherwise-healthy brain. Contradicts "optional never fails a fresh install".
**Fix:** block only on mandatory `broken` (and mandatory `unknown`); an optional `broken` becomes a loud
non-blocking warning. Update the gate's callers to still PRINT optional-broken loudly (so it's visible),
just not exit 1.
**Test:** `health-check-gate.test.mjs` — optional module `broken` → NOT a blocker; mandatory `broken` →
blocker; mandatory `unknown` → blocker; optional `unknown` → not a blocker (existing). RED first.

### #4 — `installer.mjs:786` 🟠
**Bug:** the embedder-ready post-flight now calls only `health_check` (no `expectTools`); the 4-tool
assertion (`search_vault, get_document, list_documents, vault_stats`) survives only in the not-ready
branch. A future regression dropping `get_document`/`list_documents`/`vault_stats` passes green.
**Fix:** in the ready branch, keep a structural `expectTools: EXPECT_TOOLS` check (a smoke or a tools-list
assertion) ALONGSIDE the health_check gate — both must be green. Reuse `smokeTestMcp` with `expectTools`
(already imported) before/after the gate, or assert the tool list the round-trip already returns.
**Test:** installer is integration-heavy; add/extend a focused test on the post-flight helper if one exists,
else assert via the smoke seam that a missing tool fails the gate. If no seam, at minimum add a regression
guard that EXPECT_TOOLS is still checked on the ready path. Keep it deterministic.

### #5 — `scripts/health-probe-run.mjs:71-72` 🟡
**Bug:** header promises "Fail-open: ALWAYS exit 0", but `JSON.parse(readFileSync(engine-manifest.json))`
and `.mcp.json` run synchronously at the top of the main block, OUTSIDE the promise chain. A
missing/corrupt/partially-written file (mid-update) throws uncaught → child exits non-zero, verdict cache
never refreshed (silently, detached).
**Fix:** move those reads inside the try/promise (or wrap the whole main body) so any throw routes to
`process.exit(0)`. Keep the parsing in the I/O glue, not the unit-tested `runProbeChild`.
**Test:** the main glue is "not unit-tested" by design; if cheaply testable, add a guard that a corrupt
manifest yields exit 0 + no verdict write. Otherwise verify empirically (run the child against a corrupt
`engine-manifest.json`, assert exit 0) and record the evidence in the commit message.

### #6 — `scripts/lib/reconcile-brain.mjs:117-141` 🟡
**Bug:** the seed (2.quater) precedes `runInstall`/`runReindex`; if either throws after the seed, the note
is on disk but unindexed. A retry skips re-seed (note now present) AND skips reindex (`needsReindex` false,
`seededHealthNote` false) → durable false `broken` at full depth (MCP `health_check`).
**Fix (pick the simplest that holds):**
  (a) make the reindex decision also fire when the health note is present-but-not-in-index (needs an
      index-membership seam — heavier), OR
  (b) seed the note INSIDE the same step as / immediately before the guaranteed reindex and make the pair
      transactional-ish (only mark `seededHealthNote` after the copy AND ensure the reindex always runs when
      the note was copied), OR
  (c) the pragmatic one: treat "health note on disk" as a reindex trigger independent of the seed flag —
      i.e. if `existsSync(destNote)` and a cheap marker says it's not indexed, reindex. Discuss with Thomas;
      avoid over-engineering for an unproven failure (cf. thomas-working-style) — option (b) is likely enough.
**Test:** `reconcile-brain.test.mjs` — simulate `runReindex` throwing on the seeding run, then a second
reconcile, and assert the note ends up indexed (or that a reindex is requested on the retry). RED first.

### #7 — `scripts/lib/obsidian-register.mjs:26` 🟡
**Bug:** idempotency keys on our SHA-derived id; a path the user already registered via "Open folder as
vault" lives under Obsidian's OWN random id → `!vaults[id]` true → second entry for the same path →
vault appears twice in the switcher.
**Fix:** before inserting, check whether ANY existing entry has `path === vaultPath` (normalised); if so,
no-op (return config unchanged so the I/O wrapper detects "already-registered"). Keep the pure-builder shape.
**Test:** `obsidian-register.test.mjs` — config already has `{ "<randomid>": { path: vault } }`; calling
`addVaultToObsidianConfig` returns an EQUAL config (no duplicate), and `registerVaultInObsidian` → reason
`already-registered`, no write. RED first.

### #8 — `rag/src/lib/notify.ts:30,42` 🟡
**Bug:** `buildNotifyCommand` interpolates `body`/`title` straight into `display notification "${body}"`
(osascript) and `ShowBalloonTip(..., "${body}")` (PowerShell) with no escaping. v3.3.0 newly routes
health-check `detail` (incl. arbitrary spawn-error text via `reachability` → `res.error`) into the body
(`notify-cli.ts` ← `health-probe-run.mjs:94`). A `"` breaks the AppleScript/PS string literal → notifier
exits non-zero → swallowed → lost toast exactly when it matters. (Not shell injection — args are an array.)
**Fix:** escape `"` and `` ` `` (and backslash) in `body`/`title` per target: AppleScript → `\"`; PowerShell
double-quoted string → `` ` ``-escape `"` and `` ` ``. Keep `notify-send` (Linux, array arg) as-is. Pure
function, easy to unit-test.
**Test:** `notify.test.ts` — `buildNotifyCommand("darwin", { title, body: 'a "quote" b' })` produces a
command whose arg has the quote escaped (no premature literal close); same for win32. RED first.
