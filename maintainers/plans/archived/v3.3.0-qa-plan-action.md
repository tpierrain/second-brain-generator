### F-B2 — design direction + state-of-the-art study (2026-06-21)

> QA is **PAUSED** on this finding: the fix touches enough surface that we stepped back to
> align with the state of the art before writing code. Working tree is clean; nothing started.

**State-of-the-art verdict.** What we built is the canonical **desired-state reconciliation /
convergence loop** (Kubernetes controllers, GitOps Argo/Flux, Terraform plan→apply, Chef/Puppet
convergence, Microsoft DSC Get/Test/Set, Windows Installer self-healing). We already implement it
cleanly and **level-triggered**: `reconcileBrain` (the reconciler, idempotent) converges the on-disk
state to the **desired state** = `engine-manifest.json` (`target`), runs at SessionStart reading the
*current* state (not replaying events), gated by `self-heal-detect.mjs` (drift gate). We are NOT
reinventing the wheel; the only real defect is **unfocused terminology** (code says `reconcile`, prose
says "converge" — the industry's two names for the same thing → pick one per usage).

**Decision on the mechanism (A, validated):** merge the engine-owned SessionStart hooks into
`settings.json` additively inside `reconcileBrain` (add-if-absent, dedup by command path, never touch
user entries, fail-soft) — same pattern as the existing `reconcileMcpServers`. Needs an ADR 0026
amendment (narrow the SACRED boundary to allow *engine-owned hook entries only*).

**Why the "first update runs old code" worry dissolves — the bootstrap problem.** The classic answer
is a **continuously-running, level-triggered reconciler that self-corrects on the next tick** (chef-
client every 30 min, Argo continuous). **Our tick = SessionStart.** So running the reconciler (a true
no-op when converged) at every session start is **not a hack — it is the canonical continuous-
reconciliation tick.** This makes the restart-driven convergence principled. The only already-wired
SessionStart anchor on a pre-3.2 brain is `session-status` (its code is `merge`-regime → swapped on
update 1), so for the one-time pre-3.2 → v3.3.0 jump it is the bootstrap tick; from v3.3.0 onward
auto-finalize converges in-band (1 update + 1 restart) and a new install gets all 4 hooks from the
template.

**UX target (deterministic AND pleasant):** the cheap wiring (hooks/skill/`.mcp.json`/launchers) is
deterministic + synchronous; only `npm install` stays background (irreducible). One-time pre-3.2 jump:
`/update-engine` → restart 1 (session-status converges + emits the line) → restart 2 (active, silent).
**User-facing message (Thomas's wording, FR rendering):** _"Je viens d'activer l'auto-réparation du
moteur de ton second cerveau."_ + "redémarre une dernière fois — c'est exceptionnel, les futures MAJ
s'appliqueront en un seul passage." (Source string in English per the language rule; runtime hooks are
English-only today — confirm whether to localize.)

**Terminology to lock (cloud-native 2020s):**
- **Mechanism / code →** `reconcile` / "the reconciler" (k8s + GitOps + Terraform + our `reconcileMcpServers`). Drop "the converger" as a component name.
- **Runtime / user →** `self-heal` / "auto-réparation" (Argo + MSI + `session-self-heal`).
