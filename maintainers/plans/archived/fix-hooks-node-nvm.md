# Plan — Fix: silent hooks when `node` comes from nvm (minimal PATH)

> **State: ✅ SHIPPED** (2026-06-07) — **resolved by the plan `harden-run-node-smoke-and-coverage.md`**
> (same root cause). The 3 hooks in `.claude/settings.json.template` now go through
> `{{NODE}}` → the self-heal wrapper `scripts/run-node.*` (re-resolves `node` on every run,
> covering nvm/asdf/volta/Homebrew/Windows), smoke-tested at install time on an impoverished PATH
> (loud failure). This plan remains as **context/history**; nothing more to execute.

## Origin (field feedback — Achille's bare Mac, 2026-06-07)

Install tested on a fresh Mac (account `ap`). Nominal install journey OK
(Node installed via **nvm**, non-interactive bootstrap, brain created, RAG canary OK,
demo cleanup OK, private GitHub backup OK). **BUT** a major bug was discovered:

### Root cause (confirmed)
The Claude Desktop app (Code tab) launches the **hooks** in a
**non-interactive shell with a minimal PATH** (measured: `PATH=/usr/local/bin`, without the
nvm/asdf shims nor `/opt/homebrew/bin`). Yet the 3 hooks call `node …` **directly**:

- `.claude/settings.json.template` l.42 — `statusLine` → `node "…/scripts/status-line.mjs"`
- l.52 — `PostToolUse Write|Edit` → `node "…/scripts/auto-commit.mjs"`
- l.64 — `SessionStart` → `node "…/scripts/session-status.mjs"`

If `node` comes from nvm/Homebrew, it is **not found** in this minimal PATH → the 3
hooks **fail SILENTLY**. Worst of all: **auto-commit never runs** →
notes get written but never committed (central promise broken, without a sound).
This is exactly the "silent failure" anti-pattern the project fights.

### The solution already exists at 80% in the codebase
`scripts/lib/rag-launcher.mjs` already resolves **this same root cause** for the
MCP RAG server: `rag/launch.sh` (generated by the bootstrap) prepends the usual node
locations to PATH **before** launching the server. Excerpt (sh):
```sh
add() { [ -d "$1" ] && PATH="$1:$PATH"; }
add /usr/local/bin
add /opt/homebrew/bin
add "$HOME/.asdf/shims"
for d in "$HOME"/.nvm/versions/node/*/bin; do add "$d"; done
export PATH
exec npx tsx rag/src/index.ts
```
Portable, **no machine path baked in** (only prepends existing directories),
nvm glob included. The Windows `.cmd` does the equivalent (`buildCmdLauncher`).
`applyRagLauncher(mcp, platform)` rewires `.mcp.json` to `/bin/sh rag/launch.sh`
(posix) or `cmd /c rag\launch.cmd` (win32).

> **The fix = generalize this proven mechanism to the hooks**, NOT invent an
> ad-hoc wrapper. (A test brain had improvised an ad-hoc `run-node.sh` in ITS
> instance; we don't reuse it — we do it cleanly and DRY in the source launcher.)

## Decisions made (validated with Thomas)
1. **Fail-loud guardrail INCLUDED in this lot** (step 4).
2. **Windows support NOW** (`run-node.cmd` too), for consistency with
   `rag-launcher` which already handles win32. No debt.

## Guiding principle
Reuse the already-tested PATH self-heal of `rag-launcher.mjs` (DRY, portable, proven
in the field — the RAG works on Achille's Mac). Invoke the launchers via
`/bin/sh script` (like the RAG) → **no exec bit to manage** (avoids chmod/portability).

## Steps (TDD, baby-steps — one test at a time)

### 1. Factor out the PATH self-heal block — `scripts/lib/rag-launcher.mjs`
- Extract the PATH self-heal into reusable helpers:
  `pathPrependSh()` (the `add …` lines) and `pathPrependCmd()` (the `if exist …` block).
- `buildShLauncher`/`buildCmdLauncher` reuse them → **identical behavior**
  (the existing tests `scripts/lib/rag-launcher.test.mjs` stay green = the refactor's
  safety net). First verify these tests exist and pass.

### 2. Generic `node` launchers for the hooks — same module, same helpers
- `buildNodeRunnerSh()` = header + `pathPrependSh()` + `exec node "$@"`.
- `buildNodeRunnerCmd()` = header + `pathPrependCmd()` + `node %*`.
- **Test-first**: assert that the output contains the self-heal **and**
  `exec node "$@"` / `node %*` (mirror of the RAG tests).

### 3. Generate the launchers + rewire the hooks — `bootstrap.mjs` (around l.242)
- Write `scripts/run-node.sh` and `scripts/run-node.cmd` into TARGET (like
  `rag/launch.*` at l.242-243).
- Add a `{{NODE}}` replacement resolved **per `process.platform`** (object
  `replacements` l.207):
  - posix → `/bin/sh "{{PROJECT_ROOT}}/scripts/run-node.sh"`
  - win32 → `cmd /c "{{PROJECT_ROOT}}\\scripts\\run-node.cmd"`
  - ⚠️ substitution order: `{{NODE}}` contains `{{PROJECT_ROOT}}` → make sure
    the `gen()` loop (l.226, `.split().join()` on each key) resolves both
    (put `{{PROJECT_ROOT}}` AFTER `{{NODE}}` in the iteration, or do 2
    passes). To be verified/tested explicitly.
- In `.claude/settings.json.template`, replace the 3 `node "…/X.mjs"` (l.42/52/64)
  with `{{NODE}} "…/X.mjs"`.

### 4. "Fail-loud" guardrail — `scripts/session-status.mjs`
- The SessionStart banner will now run reliably (via the wrapper).
- Add: if the working tree contains **uncommitted vault changes**
  (= a previous auto-commit didn't run), **shout** in the banner
  (⚠️ red instead of the ✅ "Repo up to date", with a prompt to check the hooks).
- Turns a future silent auto-commit failure into a **visible alert** at the
  next startup. (Read the current state of `session-status.mjs` before editing.)

### 5. Install-time verification — `bootstrap.mjs` (principle "the script judges itself")
- After generating the launchers: smoke-test `run-node.sh -e "process.exit(0)"`.
- If non-zero → **loud install failure** (non-zero exit), not a mere warning.
  Consistent with `verify-rag.mjs` and the "don't pretend" stance.

### 6. Tests & docs
- Suite green: `node --test scripts/lib/*.test.mjs scripts/*.test.mjs` (refactored
  helpers + new builders).
- Behavioral test of the wrapper: run `run-node.sh` with a PATH **stripped of
  node** but a fake directory `~/.nvm/versions/node/vX/bin` populated → must resolve;
  node truly absent → non-zero exit (no silence).
- Note in `DEVELOPING.md` / `SETUP.md`: why the hooks go through
  `run-node.*` (minimal PATH of the desktop app + node via nvm).

  principle of `rag-launcher`, and would break at the next `nvm install`).
